Weighted Scoring — How /route Picks a Backend
score =
  STATUS_WEIGHT  # green=0, yellow=50, red=100
  + kv_cache_usage × 40  # 0.0→1.0 scaled to 0→40
  + queue_depth × 3   # each queued req adds 3 pts
  + gpu_util × 0.1  # 0→100 scaled to 0→10

lowest score wins → gets the next request

Why this order? Status is the hard gate — a red backend gets +100 points penalty, making it a last resort. KV cache is the strongest continuous signal because when it fills up (>0.9), vLLM starts rejecting requests — you want to route away before that happens. Queue depth directly affects latency for the next request. GPU utilization is the weakest signal because high util with low queue just means the GPU is working efficiently — that's fine.
